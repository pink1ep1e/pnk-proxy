# 💎 PNK Telegram Proxy

MTProto FakeTLS прокси для Telegram — автоматическая установка одной командой на любой Linux-сервер.

## ⚡ Быстрый старт

```bash
bash <(curl -s https://raw.githubusercontent.com/pink1ep1e/pnk-proxy/main/setup_mtproxy.sh)
```

Через 30 секунд вы получите готовую ссылку вида:
```
tg://proxy?server=1.2.3.4&port=443&secret=ee...
```


## Что такое PNK Proxy

PNK Proxy — это MTProto прокси-сервер для Telegram с поддержкой **FakeTLS** шифрования. Трафик маскируется под обычный HTTPS, что делает прокси устойчивым к блокировкам.

**Особенности:**
- ◼ Установка в одну команду — без ручной настройки
- ◼ FakeTLS — трафик выглядит как обычный HTTPS, DPI не определяет
- ◼ Автозапуск при перезагрузке сервера через systemd
- ◼ Поддержка x86_64, ARM64, ARMv7
- ◼ Работает на Ubuntu 20.04 / 22.04 / Debian 11 / 12

## Установка

### 2. Запустите установщик

```bash
bash <(curl -s https://raw.githubusercontent.com/pink1ep1e/pnk-proxy/main/setup_mtproxy.sh)
```

Скрипт автоматически:
1. Определит ваш публичный IP
2. Установит зависимости
3. Скачает движок mtg
4. Сгенерирует уникальный FakeTLS secret
5. Откроет порт 443 в файрволе
6. Создаст systemd сервис с автозапуском
7. Выведет готовую ссылку

### 3. Скопируйте ссылку

После установки в терминале появится:

```
◼  PNK PROXY ГОТОВ К РАБОТЕ  ◼

📡 Сервер:   1.2.3.4
🔌 Порт:     443
🔑 Secret:   ee7a3c...
🌐 Домен:    www.google.com (FakeTLS)

🔗 Ссылка для Telegram (мобильный):
tg://proxy?server=1.2.3.4&port=443&secret=ee7a3c...

🔗 Ссылка (браузер / десктоп):
https://t.me/proxy?server=1.2.3.4&port=443&secret=ee7a3c...
```

## Использование ссылки

### На телефоне (Android / iOS)

1. Скопируйте ссылку `tg://proxy?...`
2. Вставьте в браузер или нажмите на неё — Telegram откроется сам
3. Нажмите **"Подключиться"**

Или вручную:
- Telegram → **Настройки** → **Данные и память** → **Тип соединения** → **Прокси** → **Добавить прокси**
- Выберите **MTProto**
- Заполните: сервер, порт, secret

### На компьютере

1. Откройте ссылку `https://t.me/proxy?...` в браузере
2. Браузер предложит открыть в Telegram Desktop
3. Нажмите **"Подключиться"**

### Поделиться с друзьями

Просто отправьте им ссылку `https://t.me/proxy?...` в любом мессенджере — они нажмут и автоматически подключатся.

## Параметры запуска

По умолчанию используется порт `443` и маскировка под `web.max.ru`. Можно изменить:

```bash
# Синтаксис:
bash setup_mtproxy.sh [ПОРТ] [ДОМЕН]

# Примеры:
bash setup_mtproxy.sh 8443 cloudflare.com
bash setup_mtproxy.sh 80 microsoft.com
bash setup_mtproxy.sh 443 apple.com
```

> Порт `443` — самый надёжный, т.к. это стандартный HTTPS-порт и его редко блокируют.

## Управление сервисом

```bash
# Статус прокси
systemctl status pnk-proxy

# Перезапустить
systemctl restart pnk-proxy

# Остановить
systemctl stop pnk-proxy

# Включить автозапуск
systemctl enable pnk-proxy

# Логи в реальном времени
journalctl -u pnk-proxy -f
```

Конфиг с параметрами сохраняется в:
```
/opt/pnk-proxy/proxy.conf
```

## Технические детали

**Движок:** [mtg v2](https://github.com/9seconds/mtg) — современная реализация MTProto прокси на Go.

**FakeTLS** — режим работы, при котором прокси-трафик выглядит как TLS-соединение к легитимному домену (например, `web.max.ru`). Системы глубокой инспекции пакетов (DPI) видят обычный HTTPS и не блокируют соединение.

**Secret** начинается с `ee` — это признак FakeTLS режима. Обычные секреты (без `ee`) работают, но легче обнаруживаются.

**Порты для использования:**
| Порт | Описание |
|---|---|
| `443` | HTTPS — самый надёжный, рекомендуется |
| `80` | HTTP — работает, но менее надёжен |
| `8443` | Альтернативный HTTPS |

**Q: Как удалить прокси?**
```bash
systemctl stop pnk-proxy
systemctl disable pnk-proxy
rm /etc/systemd/system/pnk-proxy.service
rm -rf /opt/pnk-proxy
systemctl daemon-reload
```
